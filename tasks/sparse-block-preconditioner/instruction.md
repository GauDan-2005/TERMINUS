The `/app` tooling reads the bundled basalt, flint, shale, garnet, mica, and opal inputs and writes a combined report to `/app/output/solver-audit.json`. The per-input helper `bash /app/scripts/run-one.sh <name>` writes `/app/output/<name>-audit.json` with the same shape. The report is a top-level cases array; each case reports name, ok, iterations, final_residual, reported_residual, residual_agrees, and hit_limit, as described in `/app/docs/layout.md`.

Today the report cannot be trusted. For some inputs the two residual figures disagree with each other; for some the reported figures do not match what the same inputs yield when those quantities are recomputed independently; for some an input is not flagged settled even though an independent computation settles it cleanly; and the amount of work an input reports can be wrong. Re-running `bash /app/scripts/run-matrix.sh` on unchanged inputs is not even guaranteed to reproduce the same JSON, and a per-input report does not always agree with its slice of the combined one.

Make `bash /app/scripts/run-matrix.sh` produce a report that can be trusted: every input settled, the two residual figures in agreement, and the values matching what an independent computation from the same inputs yields. Two runs from unchanged inputs must produce equivalent JSON, and each per-input report must match its slice of the combined one.

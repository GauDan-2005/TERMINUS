The Rust workload runner under `/app` sometimes leaves its audit stalled or inconsistent with bounded queues, cancelled branches, and child work mixed. Fix it so `bash /app/scripts/run-matrix.sh` completes and writes `/app/output/executor-audit.json` for the shipped cases from the current inputs.

The JSON has a top-level cases array for amber, cobalt, drake, ember, flint, and graphite. Each case reports name, ok, finished, pending, trace, ledger, and journal. Trace is the ordered list of accepted work records, where every record carries case, task, parent, depth, lane, and turn. Ledger points to a JSONL file containing the same records in the same order. Journal points to a text file with whitespace-delimited rows in the same order: case task parent depth lane turn.

A clean audit has every case finished, has no pending records, has ok true, preserves parent/depth relationships for child work, keeps cancelled branches out of the accepted trace, and leaves the JSON, ledger, and journal in agreement. The single-case helper at `/app/scripts/run-one.sh` must obey the same contract for one shipped case as the matrix run does for all cases. Repeated logical child entries should appear once in the accepted trace, and two fresh matrix runs from unchanged inputs should parse to equivalent audit data.

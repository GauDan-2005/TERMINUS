The local snapshot utility under `/app` is leaving restored directories out of agreement with their source directories in restart, quota-period, and hardlink-heavy audit cases. Get it back to a clean audit so running `bash /app/scripts/run-matrix.sh` writes `/app/output/restore-audit.json`.

The JSON has a top-level cases array for alpha, beta, gamma, delta, epsilon, and zeta. Each case reports name, source, restored, ok, digest, groups, and accounting information. Source and restored are directory paths. Digest is an object with source and restored string fingerprints. Groups is an object with source and restored arrays of relative-path arrays; each inner array is sorted and names files sharing one hardlink backing, and each outer group list is ordered by the first path in its inner arrays.

Accounting entries contain path, bytes, and period values. Byte counts are per path, including hardlink names, rather than divided across a link group. Period values use p1 for generated files under d/ when the case has moved into the later quota period, and p0 otherwise. The audit is clean when every case has ok true and file contents, directory shape, hardlink groups, and accounting records agree.

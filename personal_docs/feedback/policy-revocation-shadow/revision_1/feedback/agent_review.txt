SUMMARY
--------------------------------------------------------------------------------

This task presents a buggy Rust policy-authorization engine that processes trace
files describing grants, delegations, revocations, and grouped checks. The
agent must identify and fix logic errors across multiple evaluation modules
(k0–k5) so that `run-matrix.sh` produces a correct, deterministic JSON audit
report. The test suite (12 functions) validates delegation-chain denials, cache
reuse, epoch monotonicity, batch_id scoping, and byte-identical reruns.

1. Test Dependencies Installed in Dockerfile
--------------------------------------------------------------------------------

File:    tbench-task/environment/Dockerfile (lines 20-22)
Problem: pytest==8.4.1 and pytest-json-ctrf==0.3.5 are installed in the Docker
         image. These are test-only dependencies that cannot be used by the
         agent's Rust solution. Per the standard format, test.sh should install
         its own dependencies via uv/pip so the image stays lean and tests are
         self-contained.

Current code:
┌─────────────────────────────────────────────────────────────────────────────┐
│  RUN python3 -m pip install --break-system-packages --no-cache-dir \        │
│      pytest==8.4.1 \                                                        │
│      pytest-json-ctrf==0.3.5                                                │
└─────────────────────────────────────────────────────────────────────────────┘

Suggested fix:
┌─────────────────────────────────────────────────────────────────────────────┐
│  # Remove the pip install block from Dockerfile entirely.                   │
│  # In tests/test.sh, add before running pytest:                             │
│  pip install --break-system-packages pytest==8.4.1 pytest-json-ctrf==0.3.5  │
└─────────────────────────────────────────────────────────────────────────────┘

Explanation: Including test-only packages in the image gives agents a subtle
hint that Python testing is expected and slightly inflates the container. The
standard pattern installs test dependencies in test.sh to keep the environment
neutral and test execution self-contained.

SUMMARY
--------------------------------------------------------------------------------

This task requires debugging a custom C heap allocator whose combined audit
drifts under fragmentation, coalescing, and growth scenarios. The agent must
identify and fix bugs in footer-offset calculation (tag.c), allocation
accounting (acct.c), incomplete memory copy (move.c), and forward-only
coalescing (list.c), then rebuild and run the harness to produce deterministic
JSON audit output for six test scenarios. The test suite uses a Python reference
model (ModelArena) to independently compute expected heap signatures, free-list
counts, and byte totals for verification.

Suggested fix:
┌─────────────────────────────────────────────────────────────────────────────┐
│  # Remove the pytest install from Dockerfile entirely.                      │
│  # In tests/test.sh, add before running pytest:                             │
│  pip install --no-cache-dir pytest==8.4.1 pytest-json-ctrf==0.3.5           │
└─────────────────────────────────────────────────────────────────────────────┘

Explanation: The agent does not need pytest to solve this task (it only needs
gcc and make). Installing test-only dependencies in the image leaks information
about the test framework and bloats the agent container unnecessarily.

--------------------------------------------------------------------------------
SUMMARY
--------------------------------------------------------------------------------

This hard debugging task requires agents to repair a local inference sandbox
consisting of broken Rust modules and a faulty Go collation program. The agent
must fix per-column scaling logic, lookup-table resolution, accumulator
semantics, and JSON serialization in Rust, then rewrite the Go alignment tool
to implement proper per-field agreement checking with prior-file comparison.
The test suite contains 14 well-documented test functions that verify token
correctness, flag propagation, agreement logic, determinism, and that the Rust
binary (not a stub) produces raw output.

--------------------------------------------------------------------------------
1. Test Dependencies Installed in Dockerfile
--------------------------------------------------------------------------------

File:    tbench-task/environment/Dockerfile (lines 21-23)
Problem: pytest and pytest-json-ctrf are installed in the Docker image rather
         than in test.sh. These are test-only dependencies that should be
         installed by the test runner script, not baked into the agent
         environment.

Current code:
┌─────────────────────────────────────────────────────────────────────────────┐
│  RUN python3 -m pip install --no-cache-dir --break-system-packages \        │
│        pytest==8.4.1 \                                                      │
│        pytest-json-ctrf==0.3.5                                              │
└─────────────────────────────────────────────────────────────────────────────┘

Suggested fix: Remove these lines from the Dockerfile and update test.sh to
install its own dependencies using the standard uv-based pattern:
┌─────────────────────────────────────────────────────────────────────────────┐
│  #!/bin/bash                                                                │
│  apt-get update && apt-get install -y curl                                  │
│  curl -LsSf https://astral.sh/uv/0.7.13/install.sh | sh                    │
│  source $HOME/.local/bin/env                                                │
│                                                                             │
│  if [ "$PWD" = "/" ]; then                                                  │
│    echo "Error: No working directory set."                                  │
│    exit 1                                                                   │
│  fi                                                                         │
│                                                                             │
│  uv venv .tbench-testing                                                    │
│  source .tbench-testing/bin/activate                                        │
│  uv pip install pytest==8.4.1 pytest-json-ctrf==0.3.5                       │
│                                                                             │
│  uv run pytest /tests/test_outputs.py -rA \                                 │
│    -o cache_dir=/tmp/pytest_cache \                                         │
│    --ctrf /logs/verifier/ctrf.json                                          │
│                                                                             │
│  if [ $? -eq 0 ]; then                                                      │
│    echo 1 > /logs/verifier/reward.txt                                       │
│  else                                                                       │
│    echo 0 > /logs/verifier/reward.txt                                       │
│  fi                                                                         │
└─────────────────────────────────────────────────────────────────────────────┘

Explanation: The standard pattern keeps test infrastructure out of the agent's
image to avoid coupling and ensure portability across harnesses. The agent does
not need pytest to solve this Rust/Go debugging task.

--------------------------------------------------------------------------------
2. Non-Standard test.sh Format
--------------------------------------------------------------------------------

File:    tbench-task/tests/test.sh (lines 1-19)
Problem: test.sh skips the standard uv installation and virtual environment
         setup, directly invoking python -m pytest. This relies on the
         Dockerfile having pre-installed pytest, making it non-portable.

Current approach: test.sh assumes pytest is already available in the system
Python and runs it directly without any dependency installation step.

Suggested fix: Adopt the standard test.sh pattern that installs uv, creates a
venv, and installs pytest (as shown in Warning 1 above).

Explanation: Without self-contained dependency management, this test.sh will
fail if run on a harness that rebuilds the image without the pytest lines, or
if the task is extracted and run independently.

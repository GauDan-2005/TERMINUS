SUMMARY
--------------------------------------------------------------------------------

This task requires repairing a Rust-based incremental index invalidation
harness that processes TypeScript workspace/module traces but currently reports
stale definition locations. The agent must diagnose and fix logic bugs across
modules r0-r5 (symbol lookup filtering, freshness detection, staleness
validation) so that running the matrix script produces a correct JSON audit
report. The test suite comprises 12 pytest functions validating workspace
switching, generated-module refresh, linked-package resolution, epoch ordering,
reuse semantics, found/missing conventions, and deterministic output.

                              WARNINGS ⚠️

--------------------------------------------------------------------------------
1. Test Dependencies Pre-Staged in Environment Directory
--------------------------------------------------------------------------------

File:    tbench-task/environment/wheels/ (directory)
Problem: The environment/wheels/ directory contains pytest and pytest-json-ctrf
         wheel files that are visible to the agent at runtime. While these are
         installed at verification time (not build time), they are test-only
         dependencies residing in the agent-accessible workspace.

Current approach: Wheels are copied into /app/wheels/ by the Dockerfile, then
installed by test.sh via pip --find-links.

Explanation: This pattern is necessitated by allow_internet = false — test
dependencies must be pre-staged for offline installation. The agent cannot
exploit these wheels to solve the task, so the risk is negligible. However,
ideally test-only artifacts would reside outside the agent workspace (e.g.,
staged to /opt/wheels/ or similar non-/app path to keep the working tree clean).

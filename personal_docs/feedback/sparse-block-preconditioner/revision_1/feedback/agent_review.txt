SUMMARY
--------------------------------------------------------------------------------

This task presents a broken sparse-matrix preconditioned conjugate gradient
(PCG) solver in C++ and requires the agent to diagnose and fix four distinct
bugs across the codebase: incorrect column mapping in BSR layout assembly, a
no-op permutation function, a mis-indexed preconditioner application, and a
spurious offset in the norm computation. The solution patches four .cpp files
and rebuilds/reruns the audit scripts. The test suite (13 tests) validates
convergence, residual agreement, determinism, and per-case correctness for
six bundled input scenarios.

--------------------------------------------------------------------------------

File:    tbench-task/environment/Dockerfile (lines 21-23)
Problem: pytest==8.4.1 and pytest-json-ctrf==0.3.5 are installed in the Docker

Current code:
┌─────────────────────────────────────────────────────────────────────────────┐
│  RUN python -m pip install --no-cache-dir \                                 │
│      pytest==8.4.1 \                                                        │

└─────────────────────────────────────────────────────────────────────────────┘

Suggested fix:
┌─────────────────────────────────────────────────────────────────────────────┐
│  # Remove the pip install block from the Dockerfile entirely.               │
│  # Instead, add to tests/test.sh before the pytest invocation:              │
│  pip install pytest==8.4.1 pytest-json-ctrf==0.3.5                          │
└─────────────────────────────────────────────────────────────────────────────┘

Explanation: Test dependencies should live in test.sh so the agent workspace
does not contain tooling unrelated to its task. This also aligns the task
with the standard Terminal-Bench test runner pattern.

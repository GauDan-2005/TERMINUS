The local turn-based grid engine under `/app` produces resume audits where opposing sides step through blocked tiles or act from positions the human cannot currently see, but uninterrupted runs stay clean. Running `bash /app/scripts/run-matrix.sh` writes `/app/output/resume-audit.json`. Bundled layout maps under `/app/data`, action scripts under `/app/data/scripts`, and tables under `/app/config` define the inputs each case name uses; a correct audit reconciles both execution paths against those inputs.

The audit JSON has a top-level cases array for rook, bishop, knight, lancer, sentinel, and warden. Each case uses name, ok, fresh, resume, moves, visibility, effects, records, and log. The fresh and resume summaries describe the same scenario through the two execution paths and carry illegal, steps, and sig. The sig value is the engine signature over the resolved move list for that path, not a new per-run identifier. Each move entry carries turn, side id, from coordinates, to coordinates, kind, and legal.

Records points to a JSONL file whose nonblank rows parse to the same list in the same order. Log points to whitespace-delimited rows in the same order. The audit is clean when every case has ok true, both summaries report illegal 0, signatures match when both paths ran opposing turns, and the JSON, records file, and log file all describe the same resolved rows.

The matrix run also leaves `/app/bin/rogctl` executable. Running `bash /app/scripts/one-case.sh <case>` writes `/app/output/single-<case>.json` with the same top-level cases wrapper and the one requested row, matching that row from the matrix audit.

SUMMARY
--------------------------------------------------------------------------------

This task presents a Go/C roguelike turn-based grid engine with intentional
bugs across multiple packages (visibility cache, effect synchronization, AI
planning, legality checking, resume-from-save logic). The agent must identify
and fix these bugs so that running the matrix script produces a clean resume
audit where fresh and restored execution paths produce identical, zero-illegal
move sequences for six scenarios. The test suite validates audit structure,
per-case correctness, wall collision rules, fog-of-war visibility, timed
effects, sidecar file ordering, and trace alignment.

1. Test Dependencies Installed in Dockerfile
--------------------------------------------------------------------------------

File:    tbench-task/environment/Dockerfile (lines 21-23)
Problem: pytest==8.4.1 and pytest-json-ctrf==0.3.5 are test-only dependencies
         installed in the Docker image. These should be installed in test.sh
         to maintain separation between the task environment and test harness.

Current code:
┌─────────────────────────────────────────────────────────────────────────────┐
│  RUN python -m pip install --no-cache-dir \                                 │
│      pytest==8.4.1 \                                                        │
│      pytest-json-ctrf==0.3.5                                                │
└─────────────────────────────────────────────────────────────────────────────┘

Suggested fix: Remove from Dockerfile and install in tests/test.sh instead:
┌─────────────────────────────────────────────────────────────────────────────┐
│  #!/bin/bash                                                                │
│  apt-get update && apt-get install -y curl                                  │
│  curl -LsSf https://astral.sh/uv/0.7.13/install.sh | sh                    │
│  source $HOME/.local/bin/env                                                │
│  uv pip install pytest==8.4.1 pytest-json-ctrf==0.3.5                       │
│  python -m pytest /tests/test_outputs.py -rA \                              │
│    --ctrf /logs/verifier/ctrf.json                                          │
│  ...                                                                        │
└─────────────────────────────────────────────────────────────────────────────┘

Explanation: Test-only dependencies should not be in the agent's container
image. The agent is fixing Go code and has no use for pytest. Keeping test
deps in test.sh ensures the harness controls the test environment
independently of the task image.

2. Instruction Describes Behavior Not Covered by Tests
--------------------------------------------------------------------------------

File:    tbench-task/instruction.md (line 7)
Problem: The instruction states that "Running bash /app/scripts/one-case.sh
         <case> writes /app/output/single-<case>.json with the same top-level
         cases wrapper and the one requested row, matching that row from the
         matrix audit." No test verifies this single-case execution path.

Current approach: Tests only exercise run-matrix.sh and validate the full
audit. The one-case.sh script behavior is described as a requirement but
never verified.

Suggested fix: Add a test that runs one-case.sh and confirms its output
matches the corresponding row from the matrix audit:
┌─────────────────────────────────────────────────────────────────────────────┐
│  def test_one_case_matches_matrix(matrix_data):                             │
│      """Single-case output must match matrix audit row."""                   │
│      proc = subprocess.run(                                                 │
│          ["bash", str(APP / "scripts" / "one-case.sh"), "rook"],            │
│          cwd=APP, text=True, capture_output=True, timeout=60)               │
│      assert proc.returncode == 0                                            │
│      single = json.loads(                                                   │
│          (APP / "output" / "single-rook.json").read_text())                 │
│      matrix_row = case_by_name(matrix_data, "rook")                         │
│      assert single["cases"][0]["fresh"] == matrix_row["fresh"]              │
└─────────────────────────────────────────────────────────────────────────────┘

Explanation: Every behavior described in the task instructions should be
verified by at least one test to ensure complete coverage and prevent
partial solutions from passing.

1. Instruction Could State the Agent's Objective More Explicitly
--------------------------------------------------------------------------------

File:    tbench-task/instruction.md (lines 1-8)

Current approach: The instruction describes what a correct audit looks like
and what a clean run means, but never explicitly tells the agent that the
codebase contains bugs that need fixing. The agent must infer this by
running the matrix and observing failures.

Rationale: While acceptable for a "hard" task, adding one sentence such as
"The current codebase produces an incorrect audit; identify and fix the
issues so that the matrix run is clean" would improve clarity without
reducing difficulty, since the real challenge is locating and fixing the
bugs across multiple Go/C files.

The local snapshot utility under `/app` spans a Go staging and restore service, a native directory-measurement helper, and shell drivers that compile the workspace before running audits. Restored directories are out of agreement with their sources across all six matrix cases (alpha, beta, gamma, delta, epsilon, and zeta). Get the pipeline back to a clean audit so running `bash /app/scripts/run-matrix.sh` writes `/app/output/restore-audit.json`.

The JSON has a top-level cases array for those six names. Each case reports name, source, restored, ok, digest, groups, and accounting. Source and restored are directory paths. Digest is an object with source and restored string fingerprints. Groups is an object with source and restored arrays of relative-path arrays; each inner array is sorted and names files sharing one hardlink backing, and each outer group list is ordered by the first path in its inner arrays.

Accounting is a flat array at the case level (not nested like groups): each entry has path, bytes, and period. Byte counts are per path, including hardlink names, rather than divided across a link group; each bytes value must match that path's on-disk file size in the corresponding source or restored directory. Period values use p1 for generated files under d/ when the case has moved into the later quota period, and p0 otherwise.

A case is clean only when ok is true and file contents, directory shape, hardlink groups, and accounting records all agree. The ok flag must reflect that full contract, not digest and hardlink agreement alone. Cases that replay staged rows across a restart, such as gamma, must end with the restored layout matching the source layout in every file's final content, not merely in directory shape.

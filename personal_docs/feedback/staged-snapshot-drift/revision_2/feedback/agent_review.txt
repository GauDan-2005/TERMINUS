SUMMARY
--------------------------------------------------------------------------------

This task requires debugging a Go/C snapshot-restore utility that stages
directory trees into rows, materializes them into restored copies, and emits a
JSON audit report. The agent must identify and fix multiple subtle bugs spread
across four Go source files (core.go, replay.go, roll.go, pair.go) so that all
six matrix cases (alpha through zeta) pass a full-contract audit of content,
hardlink groups, and accounting records. The test suite runs the full matrix
pipeline and validates each case against independent filesystem observations.

1. Instruction Could Clarify the Debugging Nature of the Task
--------------------------------------------------------------------------------

File:    tbench-task/instruction.md (lines 1-8)

Current approach: The instruction states "Restored directories are out of
agreement with their sources" but does not explicitly state that there are bugs
in the Go source code that need fixing, or which subsystems are affected.

Suggested improvement: For a hard-difficulty debugging task this level of
ambiguity may be intentional, but adding one sentence such as "The source code
under /app/internal contains logic errors that cause incorrect staging,
replay, grouping, and accounting behavior" would help scope the search without
giving away the answer.

Rationale: The current phrasing relies on the agent inferring that "out of
agreement" means "buggy code" rather than missing data or misconfiguration.
A minimal hint still preserves difficulty while reducing wasted time on wrong
hypotheses.
